## 1. 模式概述

### 什么是中心化 Swap 服务

由于 TBC Pool 交易有**频率限制**，中心化 Swap 服务应运而生：
- 从 Pool 批量获取流动性
- 通过连续的 FT_Transfer 分发给用户
- 绕过 Pool 的频率限制，提升交易速度

---

## 2. 交易结构特征

### 2.1 典型流程

```
[Pool_Swap] 从池子换出 FT
    ↓
[中心化服务地址] 持有 FT
    ↓
[FT_Transfer] 发送给用户A + 找零给自己
    ↓
[FT_Transfer] 发送给用户B + 找零给自己
    ↓
... (重复N次)
    ↓
[FT_Transfer] 发送给用户N + 找零给自己
```

### 2.2 关键特征

| 特征 | 说明 |
|------|------|
| **连续转移** | 同一地址多次出现作为输入和输出 |
| **找零模式** | 每次交易都有剩余金额返回给自己 |
| **无 Pool 标记** | 中间交易不含 Pool 识别标记（无 `32436f6465` "2Code" 或 `31436f6465` "1Code"）|
| **长转移链** | 可能达到数百甚至上千层 |

## 4. 技术实现

### 4.1 为什么需要这种模式

**Pool 的限制:**
- 频率限制: 不能频繁调用 Pool 合约
- 成本较高: Pool 交易需要更多 Gas
- 复杂度: Pool 交易需要多个输入

**中心化 Swap 的优势:**
- 批量获取流动性
- 快速分发给用户
- 降低用户交易成本
- 提升用户体验

### 4.2 如何识别这种模式

**代码检测:**
```python
# 检查是否是中心化 Swap 模式
def is_centralized_swap_pattern(tx_chain):
    """
    特征:
    1. 长转移链 (>100层)
    2. 同一地址多次出现
    3. 无 Pool 标记
    4. 连续的 FT_Transfer
    """
    if len(tx_chain) < 100:
        return False
    
    # 检查地址重复
    addresses = set()
    for tx in tx_chain:
        for addr in tx.get_addresses():
            addresses.add(addr)
    
    # 如果地址很少但交易很多，可能是中心化服务
    if len(addresses) < 5 and len(tx_chain) > 500:
        return True
    
    return False
```
