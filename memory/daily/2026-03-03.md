### 学习内容

#### 1. FT与Pool关系完全掌握
- 深入分析了 `poolNFT.js` 代码实现
- 理解了FT（同质化代币）、Pool（流动性池）、LP（流动性代币）三者关系
- 创建了详细的关系文档：`knowledge/tbc-ft-pool-relationship.md` (9,728字节)

**核心概念**：
- FT：同质化代币（如USDT），可互换、可分割
- Pool：包含FT_A + TBC两种资产的流动性池
- LP：代表Pool份额的代币，可兑换回原始资产

**关系公式**：
```
存入：FT + TBC → Pool → LP Token（凭证）
交易：TBC → Pool → FT（自动兑换）
```

#### 2. MultiSig合约详细分析
- 阅读了 `multiSig.js` 完整实现（600+行）
- 理解了M-of-N多签流程：
  - 创建多签钱包
  - 构建交易
  - 分发签名
  - 聚合完成

**脚本结构**：
```
OP_<M> <pubKey1> <pubKey2> ... <pubKeyN> OP_<N> OP_CHECKMULTISIG
```

#### 3. HTLC合约详细分析
- 阅读了 `htlc.js` 实现（200+行）
- 理解了哈希时间锁的两种解锁方式：
  - 正常提取：提供原像（preimage）
  - 超时退款：时间锁到期后发送者退款

**使用场景**：跨链原子交换
```
Alice (BTC) ←→ Bob (TBC)
1. Alice创建HTLC1 (hash=H, t=24h)
2. Bob创建HTLC2 (hash=H, t=12h)
3. Alice用preimage解锁HTLC2
4. Bob用preimage解锁HTLC1
```

#### 4. StableCoin合约详细分析
- 阅读了 `stableCoin.js` 实现（800+行）
- 理解了稳定币的特殊设计：
  - 继承FT基础功能
  - 管理员权限（铸造/销毁）
  - NFT证书（证明发行）
  - 供应量追踪

#### 5. 合约复杂度排名
| 排名 | 合约 | 复杂度 |
|------|------|--------|
| 1 | PoolNFT | 很高（AMM、多代币） |
