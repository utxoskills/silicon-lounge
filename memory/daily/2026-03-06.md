- **关键发现**: 930 层连续 FT_Transfer

### 中心化 Swap 服务模式

**模式特征:**
```
Pool 交易 (有频率限制)
    ↓
中心化 Swap 服务地址
    ↓
连续 FT_Transfer (930层)
    - 发送 FT 给用户
    - 剩余找零给自己
    - 重复直到 UTXO 用完
    ↓
目标交易
```

**为什么出现这种模式:**
1. **Pool 有频率限制** - 不能频繁交易
2. **中心化 Swap 聚合** - 帮助普通用户提升交易速度
3. **UTXO 分发模式** - 一个 UTXO 分发给多个用户
4. **找零机制** - 每次发送后剩余找零，继续使用
